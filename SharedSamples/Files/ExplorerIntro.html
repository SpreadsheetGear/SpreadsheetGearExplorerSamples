SpreadsheetGear Explorer Samples

The samples in Windows Forms Explorer, WPF Explorer and Web Explorer demonstrate a wide variety of APIs and features offered by the following SpreadsheetGear products:

  • SpreadsheetGear Engine for .NET - the primary SpreadsheetGear library that provides a core set of APIs to read, write, manipulate and calculate workbooks, build charts, format worksheets and cells, and more.
  • SpreadsheetGear for Windows - builds on the SpreadsheetGear Engine for .NET to add powerful Excel-compatible viewing, editing, formatting, calculating, filtering, sorting, charting, printing and more to your Windows Forms and WPF applications with the easy-to-use WorkbookView and FormulaBar controls.

Learn more about these products on our Features Page and more details on their diffences on our Comparison Page.

SpreadsheetGearExplorer Visual Studio Solution Description

The three executable Visual Studio projects (WindowsFormsExplorer, WPFExplorer and WebExplorer) utilize a set of samples that are defined primarily in the SharedSamples Visual Studio library project. This SharedSamples library utilizes two primary sample types, derived from the following base class / interface:

  • SharedEngineSample - samples deriving from this class demonstrate features that depend only on API available from the SpreadsheetGear Engine for .NET product, thereby showing how such routines can be solely executed in-memory / programmatically (i.e, no UI needs to be involved in the execution of these samples. This is demonstrated most clearly when running the Web Explorer to execute these samples in the context of an ASP.NET Core Web App. For the Windows Forms Explorer and WPF Explorer, the workbook in which a given sample is executed is displayed on a WorkbookView UI control to more easily observe the results..
  • ISharedWindowsSample - samples derived from this interface will demonstrate features that more directly involves a UI of some sort, such as the WorkbookView control. Much of the source code for these samples is still defined within the SharedSamples Visual Studio project, but supporting UI framework-specifc code is also required in the individual WindowsFormsExplorer and WPFExplorer projects. In a few cases, sample source code is not shared at all and instead entirely defined from within the WindowsFormsExplorer and WPFExplorer projects because they depend on specific features only available to their particular UI framework (such as Control Templates in XAML to stylize a WPF WorkbookView).
